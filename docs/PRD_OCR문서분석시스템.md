## 1. 개요 (Overview)

본 제품은 PDF, HWP, DOCX, XLSX 등 다양한 포맷의 문서를 입력받아 시각적 레이아웃, 텍스트, 표, 이미지 등의 구조를 원본과 동일하게 인식·추출하는 OCR 및 문서 파싱 시스템이다. 추출 결과는 후속 시스템이 데이터 유실 없이 활용할 수 있도록 정형화된 마크다운(Markdown) 및 JSON 구조로 변환된다.

기존 OCR 도구가 단순 텍스트 추출에 머무는 것과 달리, 본 시스템은 문서의 시각적 흐름과 데이터 구조(다단 레이아웃, 병합 셀, 스타일 계층 등)를 보존하는 것을 핵심 차별점으로 한다.

## 2. 문제 정의 (Problem Statement)

조직 내 문서는 PDF, 한글(HWP), 워드, 엑셀 등 이질적인 포맷으로 흩어져 있으며, 이를 디지털 데이터로 활용하려면 구조를 유지한 채 정형화하는 작업이 필요하다. 현재의 어려움은 다음과 같다.

- 일반 OCR은 텍스트는 추출하나 표·다단·계층 구조가 깨져 후속 활용이 어렵다.
- 포맷마다 구조적 특성(병합 셀, 개요 번호, 다단 등)이 달라 통합 처리가 힘들다.
- 각주·캡션·페이지 번호 등 부가 정보가 누락되어 원본 대조 시 신뢰도가 낮다.

## 3. 목표 (Goals) 및 비목표 (Non-Goals)

### 3.1 목표

본 시스템은 다중 포맷 문서를 입력받아 원본의 레이아웃과 데이터 구조를 보존한 정형 출력(마크다운/JSON)을 생성하는 것을 목표로 한다. 정확성, 레이아웃 보존, 무손실의 세 가지 원칙을 충족해야 한다.

### 3.2 비목표

문서의 의미 요약, 번역, 자동 분류, 내용 기반 질의응답 등 추출 이후의 고차원 분석은 본 단계의 범위에서 제외한다. 본 시스템은 "구조 보존형 추출"까지를 책임 범위로 한다.

## 4. 핵심 원칙 (Core Principles)

시스템 출력은 다음 세 원칙을 반드시 만족해야 한다.

1. **정확성(Accuracy):** 오탈자나 왜곡 없이 텍스트를 인식하며, 전문 용어·숫자·기호의 오인식을 최소화한다.
2. **레이아웃 보존(Layout Preservation):** 헤더, 푸터, 제목, 본문, 사이드바, 다단 구조 등 시각적 배치를 읽기 순서에 맞게 정렬한다.
3. **무손실(No Loss):** 각주, 미주, 페이지 번호를 포함한 모든 정보를 누락 없이 추출한다.

## 5. 대상 사용자 (Target Users)

본 시스템의 주 사용자는 대량의 이기종 문서를 디지털 자산화해야 하는 공공기관·기업의 문서 관리 담당자, 추출 데이터를 파이프라인에 투입하는 데이터 엔지니어, 그리고 정형화된 결과를 소비하는 후속 자동화 시스템(RAG, 검색 인덱스 등)이다.

## 6. 기능 요구사항 (Functional Requirements)

### 6.1 포맷별 인식 기능

| ID | 포맷 | 요구사항 |
| :--- | :--- | :--- |
| FR-1 | PDF / 스캔 이미지 | 다단(Multi-column) 레이아웃을 좌상→우하 읽기 순서로 분리 추출. 그림·그래프 하단 캡션을 본문과 분리하여 인용 형식으로 표기 |
| FR-2 | HWP / DOCX | 개요 번호(1., 가., (1))와 글머리 기호의 들여쓰기 계층을 마크다운 문법으로 구현. 굵게·밑줄·기울임 등 강조 스타일 변환 |
| FR-3 | XLSX | 병합 셀의 값을 대표 셀에 배치하고 확장 구조를 표로 표현. 다중 시트는 시트명 단위로 구분 처리 |

### 6.2 데이터 요소별 출력 기능

| ID | 요소 | 요구사항 |
| :--- | :--- | :--- |
| FR-4 | 헤더·텍스트 | 제목 계층 구조에 따라 `#`~`####`를 엄격히 구분 적용 |
| FR-5 | 표(Table) | 모든 표를 마크다운 표로 변환, 셀 내 줄바꿈은 `<br>` 처리 |
| FR-6 | 시각 요소 | 이미지·차트·다이어그램 위치에 태그 삽입 후 시각 내용 요약 기술 |

### 6.3 출력 포맷 기능

| ID | 요구사항 |
| :--- | :--- |
| FR-7 | 최종 결과를 지정된 마크다운 구조로만 출력(별도 인사말·설명 생략) |
| FR-8 | 세션/페이지/시트 단위로 제목을 구분하여 순차 배치 |
| FR-9 | 후속 시스템 연계를 위한 JSON 구조 출력 지원 |

## 7. 사용자 시나리오 (User Scenarios)

담당자가 여러 포맷이 섞인 문서 묶음을 시스템에 투입하면, 시스템은 각 파일의 포맷을 식별하고 포맷별 지침에 따라 구조를 보존한 채 추출한다. 결과물은 원본 레이아웃을 기반으로 한 마크다운으로 산출되며, 표·캡션·계층 구조가 그대로 유지되어 담당자는 별도 재정리 없이 후속 파이프라인에 전달할 수 있다.

## 8. 성공 지표 (Success Metrics)

| 지표 | 목표 |
| :--- | :--- |
| 텍스트 인식 정확도(문자 단위) | 99% 이상 |
| 표 구조 보존율 | 95% 이상 |
| 레이아웃 읽기 순서 정확도 | 95% 이상 |
| 정보 누락률(각주·캡션·페이지번호) | 1% 이하 |

## 9. 가정 및 제약 (Assumptions & Constraints)

입력 문서는 시스템이 지원하는 포맷(PDF, HWP, DOCX, XLSX 및 스캔 이미지)에 한한다. 스캔 품질이 현저히 낮은 문서는 정확도가 저하될 수 있으며, 출력은 마크다운/JSON 규격을 엄격히 준수하므로 임의 서식은 허용되지 않는다.

## 10. 향후 과제 (Future Work)

추출 결과를 활용한 의미 검색, 자동 요약, 다국어 번역, 문서 분류 기능을 후속 단계에서 검토한다. 또한 PPTX 등 추가 포맷 지원과 추출 신뢰도 점수 제공도 향후 확장 대상이다.
