# 시스템 구동 가이드

OCR 인식 시스템(`ocr-docintel`)을 설치하고 실행하는 방법을 정리한다.

## 1. 사전 요구사항

- Python 3.10 이상 (현재 검증 환경 3.11)
- 선택 사항
  - 스캔 PDF/이미지 OCR — 시스템에 **Tesseract 바이너리**(kor+eng) 설치
  - LLM 맥락 교정(Accurate 모드) — `ANTHROPIC_API_KEY` 환경변수

> Tesseract와 API 키가 없어도 전 과정이 결정론적으로 동작한다. 스캔 페이지는 건너뛰고, LLM 단계는 입력을 그대로 반환하는 폴백으로 처리된다.

## 2. 설치

```bash
pip install -e ".[dev]"            # 개발 의존성(pytest, reportlab 등) 포함
pip install -e ".[dev,llm]"       # + anthropic SDK(LLM 맥락 단계 실호출용)
pip install -e ".[dev,web]"       # + 웹 UI 의존성(fastapi, uvicorn 등)
```

웹 UI까지 한 번에 쓰려면 `pip install -e ".[dev,web]"`를 권장한다.

설치 확인.

```bash
python -c "import ocr_docintel; print('ocr_docintel OK')"
```

## 3. 구동

### 3-1. 웹 UI + API

```bash
python webserver.py
```

- 접속 주소 — http://127.0.0.1:8000
- 사용 흐름 — 문서 업로드 → 모델 선택(Fast/Balanced=결정론, Accurate=LLM 맥락) → 변환 결과(Markdown·JSON) 확인
- API — `POST /api/convert` (multipart: `file`, `use_llm`) → `{markdown, json, elements, source_format}`
- 종료 — 터미널에서 `Ctrl+C`

### 3-2. CLI 변환

```bash
python -m ocr_docintel <문서> -o <출력_디렉터리>            # LLM 맥락 단계 포함
python -m ocr_docintel <문서> -o <출력_디렉터리> --no-llm   # 결정론적 폴백만
```

출력은 변환된 `*.md`와 `*.json`이다. 지원 포맷은 텍스트 PDF, 스캔 PDF/이미지(PNG·JPG), DOCX, XLSX, HWP이다.

## 4. 테스트로 동작 확인

```bash
pytest -q                          # 전체 테스트
pytest tests/test_pipeline.py -q   # E2E + KPI 게이트만
```

OCR·HWP·실 LLM 호출 테스트는 바이너리·키·샘플이 없으면 자동으로 skip된다.

## 5. 자주 막히는 지점

- `ModuleNotFoundError: No module named 'ocr_docintel'` — 패키지 미설치. 2장 설치를 먼저 수행한다.
- 스캔 PDF/이미지가 빈 결과 — Tesseract 미설치. `TESSERACT_CMD` 환경변수 → PATH → `C:\Program Files\Tesseract-OCR` 순으로 자동 탐색한다.
- Accurate 모드가 결정론 결과와 동일 — `ANTHROPIC_API_KEY` 미설정 시 폴백된 정상 동작이다.
