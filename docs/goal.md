## 1. 한 줄 목표

다양한 포맷의 문서를 입력받아 **시각적 레이아웃·텍스트·표·이미지 구조를 원본과 동일하게 보존**한 정형 출력(Markdown + JSON)으로 변환한다.

## 2. 핵심 원칙 (불변)

PRD §4 / System Prompt §2 에서 도출한 세 원칙은 모든 산출물의 합격 기준이다.

1. **정확성(Accuracy)** — 오탈자·왜곡 없이 텍스트 인식, 전문용어·숫자·기호 오인식 최소화.
2. **레이아웃 보존(Layout Preservation)** — 헤더/푸터/제목/본문/다단 구조를 읽기 순서에 맞게 정렬.
3. **무손실(No Loss)** — 각주·미주·페이지번호 등 모든 정보를 누락 없이 추출.

추가 설계 원칙(개발계획서 §4.1): **LLM은 "근거 좌표가 있는 텍스트"만 다룬다.** 원문에 없는 내용 생성을 구조적으로 차단(grounded extraction).

## 3. 성공 지표 (Success Metrics)

PRD §8 와 개발계획서 §2.2 KPI 를 MVP 수준으로 조정한 검증 기준이다. (`tests.md`에서 자동 측정)

| 지표 | 원본 목표 | MVP 합격선 | 측정 방법 |
| :--- | :--- | :--- | :--- |
| 텍스트 인식 정확도(문자) | ≥ 99% (CER ≤ 3%) | 텍스트 PDF에서 CER ≤ 1% | 골든셋 대비 문자오류율 |
| 표 구조 보존율 | ≥ 95% (TEDS ≥ 0.85) | 단순표 셀 일치율 ≥ 95% | 셀 단위 비교 |
| 읽기 순서 정확도 | ≥ 95% | 단/다단 문단 순서 일치 ≥ 90% | 문단 순서 라벨 비교 |
| 정보 누락률 | ≤ 1% | 페이지번호·캡션 누락 0건 | 골든셋 요소 카운트 |
| 출력 포맷 규격 준수율 | 100% | JSON 스키마 + MD 구조 검증 100% | 스키마 검사 |

## 4. 범위 (Scope)

### 4.1 In-Scope (MVP)
- 텍스트 레이어 PDF 입력 → 구조 추출.
- 출력: System Prompt §5 마크다운 구조 + TRD §4.4 JSON 스키마.
- 요소: 제목 계층, 본문 단락, 표, 이미지/캡션 태그.
- 하이브리드: 라이브러리(PyMuPDF/pdfplumber)로 좌표·구조 확보 → Claude API로 읽기순서 교정·표 정규화·캡션 분리.
- 검증: JSON 스키마 + MD 구조 규격 검사, 골든셋 회귀.

### 4.2 Out-of-Scope (MVP 제외, 후속)
- HWP/DOCX/XLSX 파서(인터페이스만 예약), 스캔 이미지 OCR(Tesseract).
- 벡터DB/RAG 임베딩, 검수 UI, GPU 가속, 페이지 횡단 표 병합.
- 의미 요약·번역·분류(PRD §3.2 비목표).

## 5. 완료 정의 (Definition of Done)

1. `docs/`에 goal/plan/status/tests 4종 + checklist/context-notes 존재.
2. `src/`에 텍스트 PDF → MD+JSON 변환 MVP 파이프라인이 동작.
3. `tests/`가 골든셋 PDF에 대해 통과(스키마 100%, CER/표/순서 합격선 충족).
4. LLM 단계는 API 키 없이도 결정론적 폴백으로 파이프라인이 끝까지 동작(테스트 오프라인 가능).
5. `status.md`가 최신 상태를 반영.
